## 简介

**Genkicap Video Workflow v1.0** 是一个基于选题驱动的视频脚本创作工作流系统。它帮助创作者从一个选题想法出发，通过系统化的 **7 个阶段**，逐步完善并生成高质量、可执行的视频脚本。

### 实现架构

**v1.0.12**: Prompt 驱动 + project-manager skill

- **Prompt 驱动**: 核心工作流逻辑定义在 `.claude/CLAUDE.md` 中
- **project-manager skill**: 自动管理项目创建、目录结构和元数据
- **灵活工作流**: 可根据视频类型跳过某些阶段（如短视频可跳过调研和大纲）

### 核心理念

```
💡 "我想讲什么内容" → 📝 系统化脚本 → 🎬 可执行制作
```

### 适用场景

- ✅ **长视频创作** (10分钟+)
- ✅ **深度内容制作** (教程、解说、评论等)
- ✅ **新手用户** (无需数据积累)
- ✅ **个人创作者** 或 **小团队**

## 完整工作流程

### 阶段0: 需求确认

**目标**: 收集核心信息，确定当前创作阶段

**收集信息**:
- 内容领域 (科技/美食/教育等)
- 当前阶段 (刚开始/已有选题/已有草稿等)
- 目标平台 (B站/抖音/小红书/YouTube)
- 视频时长 (短视频/中视频/长视频)

**输出**: 项目概况确认 → 自动跳转到对应阶段

---

### 阶段1: 选题确认 → `idea.md`

**目标**: 从零散想法生成结构化的选题描述

**执行任务**:
1. 整理用户输入
2. 提取核心主题、选题角度、目标受众、预期价值
3. 生成搜索关键词

**输出文件**: `stages/idea.md`

**示例**:
```markdown
## 选题描述

**核心主题**: 5款主流AI绘画工具横向对比测评

**选题角度**: 从价格、易用性、生成质量三个维度，
为设计师和创作者提供选购指南

**目标受众**: 想要使用AI绘画工具但不知道选哪个的设计师、
创作者、内容从业者

**预期价值**: 帮助观众了解各工具的优劣势，节省试错成本，
选择最适合自己的工具

**搜索关键词**:
- AI绘画工具对比
- Midjourney vs Stable Diffusion
- AI art generator review
```

---

### 阶段2: 选题调研 → `research.md`

**目标**: 搜索相关内容，提供 3 个不同角度和标题方案

**执行任务**:
1. 基于 `idea.md` 的关键词使用 WebSearch 搜索
2. 整理同类内容的角度和结构
3. 生成 3 个不同角度，每个角度配 2-3 个标题
4. 推荐最优方案

**输出文件**: `stages/research.md`

**示例**:
```markdown
## 选题调研报告

### 角度1: 实测对比型

**核心观点**: 用同一个 prompt 测试 5 款工具，直观对比生成效果

**推荐标题**:
1. "5款AI绘画工具实测对比 | 用同一个prompt看谁最强"
2. "AI绘画工具横评 | Midjourney真的最强吗？"
3. "花了500元测试5款AI绘画工具，结果让人意外"

**参考内容**:
- YouTube: "I Tested 5 AI Art Generators" (500K views)
- B站: "AI绘画工具大横评" (100W播放)

**优势**: 直观、数据化、可信度高
**劣势**: 制作成本较高

---

### 角度2: 新手教程型
[...]

### 角度3: 性价比分析型
[...]

### 推荐方案 ⭐

**推荐角度**: 角度1 (实测对比型)
**推荐标题**: "5款AI绘画工具实测对比 | 用同一个prompt看谁最强"
**推荐理由**:
- 数据支撑: 同类型视频平均播放量 > 50W
- 差异化: 大部分测评只对比 2-3 个工具，5个更全面
- 可执行: 成本可控，制作流程清晰
```

---

### 阶段3: 框架设计 → `frame.md`

**目标**: 设计视频的时间轴、内容层次、Hook、转折点

**执行任务**:
1. 设计时间轴和段落划分
2. 规划一级观点和二级支撑点
3. 设计 Hook (开场钩子)
4. 设计转折点 (保持观众兴趣)
5. 设计结尾 CTA (引导行动)
6. **推荐合适的模板**

**输出文件**: `stages/frame.md`

**示例**:
```markdown
## 视频框架设计

**推荐模板**: Review.md (测评类模板)
**总时长**: 8分钟

### 时间轴设计

| 时间段 | 段落名称 | 核心内容 | 时长 |
|--------|----------|----------|------|
| 0-10s | Hook | 用悬念开场："花了500元测试5款AI工具，第3个结果让我意外" | 10s |
| 10s-30s | 问题引入 | "市面上AI绘画工具那么多，到底该选哪个？" | 20s |
| 30s-2m | 测试标准 | 介绍测试方法、prompt、评判标准 | 1m30s |
| 2m-6m | 逐一测试 | 5款工具逐一展示生成效果 (每款1m) | 4m |
| 6m-7m30s | 综合对比 | 对比表格，总结各工具优劣势 | 1m30s |
| 7m30s-8m | 总结/CTA | 推荐选购建议 + 引导订阅 | 30s |

### Hook 设计思路

**Hook类型**: 数据悬念式
**具体方案**: "花了500元测试5款AI绘画工具，第3个结果让我意外..."
**预期效果**: 制造悬念，引发好奇，降低划走率
```

---

### 阶段4: 脚本草稿 → `draft.md`

**目标**: 基于框架生成详细的脚本草稿

**执行任务**:
1. 生成详细 Outline
2. 应用推荐模板的规则
3. 生成每个段落的口播文本、画面建议、设计意图

**输出文件**: `stages/draft.md`

**示例**:
```markdown
## 脚本草稿

### 段落1: Hook (0-10s)

**【口播文本】**
"花了500元，我测试了市面上5款主流AI绘画工具。
用同一个prompt生成，第3个工具的结果让我完全没想到..."

**【画面建议】**
- 快速剪辑: 5款工具的生成效果闪现
- 文字动效: "500元" 数字放大
- 在第3个工具的效果上停顿，制造悬念

**【设计意图】**
用悬念开场，引发好奇心，降低划走率

---

### 段落2: 问题引入 (10s-30s)
[...]
```

---

### 阶段5: 编辑优化 → `edit.md`

**目标**: 优化口播文本、完善画面建议、调整时长

**执行任务**:
1. 优化口播文本 (自然度、节奏、情绪)
2. 完善画面建议 (镜头、机位、特效)
3. 调整时长 (删减冗余、扩充重点)

**输出文件**: `stages/edit.md`

**优化示例**:
```
❌ 草稿版: "在当今这个快速发展的AI时代，绘画工具已经成为了..."
✅ 优化版: "现在AI绘画工具越来越火，你有没有遇到选择困难？"
```

---

### 阶段6: 用户确认 → `script.md`

**目标**: 生成最终可执行的脚本

**执行任务**:
1. 整理 `edit.md` 的内容
2. 生成最终脚本格式
3. 添加制作建议和注意事项
4. 保存到根目录 `script.md`

**输出文件**: `script.md` (最终脚本) ⭐

### 高级用法

#### 添加补充资料

在 `contexts/` 目录下添加参考资料：

```
scripts/my-video/
├── contexts/
│   ├── videos/
│   │   └── reference-video-1.md  # 参考视频分析
│   ├── research/
│   │   └── market-analysis.md    # 市场调研资料
│   └── accounts/
│       └── competitor-1.md       # 竞品账号分析
```

系统会在调研和框架设计时自动读取这些资料作为参考。

#### 版本回溯

如果需要回到之前的版本：

```
你: 我觉得第一版的框架更好，能回到那个版本吗？

助手: 当然可以！让我读取 _archive/frame_v01.md ...
[读取历史版本，展示内容]

你想要：
• 恢复这个版本 → 覆盖当前 frame.md
• 参考对比 → 保留当前版本，参考历史版本
```

#### 自定义模板

你可以基于现有模板创建自定义模板：

1. 复制 `references/templates/en-US/Base.md` 到同目录
2. 重命名为 `Custom.md`
3. 根据需求修改模板内容
4. 在框架设计阶段手动指定使用自定义模板

## 常见问题

### Q: 我可以跳过某些阶段吗？

**A**: 可以。如果你已经有了某个阶段的输出（如已有选题），告诉助手你的当前阶段，系统会自动跳转到对应位置。

### Q: 生成的脚本可以直接用于拍摄吗？

**A**: 是的！阶段6生成的 `script.md` 是可执行的脚本，包含：
- 完整的口播文本
- 详细的画面建议
- 拍摄和后期要点

你可以直接按照脚本进行拍摄和剪辑。

### Q: 如果我对某个阶段的输出不满意怎么办？

**A**: 每个阶段完成后都会征求你的确认。如果不满意，可以：
- 告诉助手需要调整的部分
- 重新生成该阶段的输出
- 回到上一个阶段重新开始

### Q: 模板是强制使用的吗？

**A**: 不是。模板是推荐性的，你可以：
- 接受推荐的模板
- 选择其他模板
- 不使用模板，完全自定义

### Q: 支持哪些平台？

**A**: v1.0 支持所有主流平台：
- 国内: B站、抖音、小红书
- 海外: YouTube、TikTok

系统会根据目标平台提供差异化的建议（如标题风格、时长控制等）。

### Q: 我可以同时创作多个视频项目吗？

**A**: 可以！在 `scripts/` 目录下创建多个项目目录即可：

```
scripts/
├── project-1/
├── project-2/
└── project-3/
```

每个项目独立管理，互不影响。

### Q: 如何更新到 v2.0 或 v3.0？

**A**: v1.0、v2.0、v3.0 是独立的版本，可以共存使用：

- **v1.0** (Mode 1 - 选题驱动): 适合长视频、深度内容
- **v2.0** (Mode 2 - 结构驱动): 适合短视频、快速起量
- **v3.0** (Mode 3 - 数据驱动): 适合数据优化、规模化

你可以根据不同场景选择使用不同版本。
