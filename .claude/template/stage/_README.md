# Mode1 模版结构说明

> 本文档说明 Mode1（选题梳理驱动）的模版文件结构和使用方式

---

## 📁 模版目录结构

```
.claude/template/stage/
├── idea.md              # 阶段1：选题沟通输出模版
├── frame.md             # 阶段2：框架搭建输出模版
├── research.md          # 阶段3：内容调研输出模版（可选）
├── outline.md           # 阶段4：大纲确认输出模版
├── draft.md             # 阶段5：脚本撰写输出模版
├── script.md            # 阶段7：最终脚本输出模版
└── README.md            # 本文件
```

---

## 📝 输出模版（Output Templates）

### 核心设计理念

所有模版都包含**完整的输出内容**和**下一步引导**，Agent 只需读取模版文件即可理解完整流程，无需额外的交互指令文件。

---

### 1. idea.md - 选题沟通输出模版

**用途**：阶段1 输出文件 `stages/idea.md`

**输出模式**：单步输出

**内容结构**：
- 选题主题/领域开场描述
- 竞品频道参考（≥2个）
- 3个核心角度（表格形式：角度序号 | 角度名称 | 核心聚焦点 | 观众目标收获）
- 标题建议（每个角度配2-3个标题）
- **下一步确认**（模版自带）

**关键要求**：
- ✅ 标题格式：`# [选题主题/领域] 视频内容策划`
- ✅ 角度必须用表格
- ✅ 结尾包含"下一步确认"引导用户选择

---

### 2. frame.md - 框架搭建输出模版

**用途**：阶段2 输出文件 `stages/frame.md`

**输出模式**：两步输出

**第一步 - 生成框架并请用户确认**：
- 开场描述（回应用户选择的角度）
- 核心概念说明
- 建议结构（结构类型说明）
- 分段框架表格（板块 | 核心聚焦点 | 预估字数 | 预估时长）
- **结尾确认问题**："你觉得这个结构和预估时长合适吗？是否可以开始创作脚本了？"

**第二步 - 下一步行动确认**：
- 关键信息汇总（角度、概念、结构、时长、字数）
- 下一步行动选项（进入大纲/直接写脚本/补充资料/调整框架）

**关键规则**：
- ✅ 单平台版（不在模版中处理多平台）
- ✅ 时长显示：<60秒用秒，≥60秒用分钟
- ✅ 字数换算：1分钟 ≈ 150-180字

---

### 3. research.md - 内容调研输出模版（可选）

**用途**：阶段3 输出文件 `stages/research.md`

**说明**：Mode1 的阶段3通常会被跳过，直接从框架进入大纲。此模版为可选。

**内容结构**：
- Executive Summary（执行摘要）
- Key Historical Context（核心历史背景）
- Major Trends（主要趋势）
- Real-World Applications（实际应用案例）
- Supporting Data and Statistics（数据统计）
- Expert Opinions（专家观点）
- Competitive Content Analysis（竞品内容分析）
- Key Takeaways（关键要点）

**调研深度**：
- 深度分析型（长视频）：完整11个模块
- 快速产出型（短视频）：精简版（6个核心模块）

---

### 4. outline.md - 大纲确认输出模版

**用途**：阶段4 输出文件 `stages/outline.md`

**输出模式**：两步输出

**第一步 - 生成大纲并请用户确认**：
- 标题和总字数
- 各板块详细大纲（每个板块3-6个要点）
- 论证逻辑和过渡衔接说明
- **结尾确认问题**："这个大纲符合你的框架要求吗？需要调整哪个板块的内容？"

**第二步 - 下一步行动确认**：
- 关键信息汇总（标题、时长、字数、各板块分配）
- 下一步行动选项（开始脚本撰写/调整大纲/补充资料）

---

### 5. draft.md - 脚本撰写输出模版

**用途**：阶段5 输出文件 `stages/draft.md`

**输出模式**：两步输出

**第一步 - 生成脚本并请用户确认**：
- 各板块完整脚本内容
- 每个板块的实际字数统计
- **结尾确认问题**："这个脚本符合你的预期吗？需要调整哪个部分？"

**第二步 - 下一步行动确认**：
- 关键信息汇总（标题、总字数、各板块字数、超出/不足说明）
- 下一步行动选项（进入编辑优化/调整某板块/补充资料/直接输出）

**生成策略**：
- 短视频（≤3分钟，约≤500字）：一次性生成
- 中等视频（3-10分钟，约500-1800字）：一次性或分段生成
- 长视频（>10分钟，约>1800字）：一次性或分段生成并确认

---

### 6. script.md - 最终脚本输出模版

**用途**：阶段7 输出文件 `script.md`（项目根目录）

**输出模式**：单步输出

**内容结构**：
- 脚本标题
- 元信息（目标时长、总字数、创作时间）
- 完整脚本内容（从 draft.md 复制）
- 脚本统计表格（板块 | 字数 | 预估时长）

**关键要求**：
- ✅ 标题格式：`# 《[视频标题]》`
- ✅ 必须包含元信息和统计表格
- ✅ 从 draft.md 复制内容，draft.md 保留作为备份

---

## 🔄 工作流程

```
阶段1: 选题沟通
↓ 使用 idea.md 模版 → 生成选题角度 + 下一步确认
↓ 用户选择角度

阶段2: 框架搭建
↓ 使用 frame.md 模版 → 第一步：生成框架 + 确认问题
↓ 用户确认
↓ 使用 frame.md 模版 → 第二步：下一步行动确认
↓ 用户选择下一步

阶段3: 内容调研（可选）
↓ 使用 research.md 模版 → 生成调研报告

阶段4: 大纲确认
↓ 使用 outline.md 模版 → 第一步：生成大纲 + 确认问题
↓ 用户确认
↓ 使用 outline.md 模版 → 第二步：下一步行动确认
↓ 用户选择下一步

阶段5: 脚本撰写
↓ 使用 draft.md 模版 → 第一步：生成脚本 + 确认问题
↓ 用户确认
↓ 使用 draft.md 模版 → 第二步：下一步行动确认
↓ 用户选择下一步

阶段6: 优化编辑（用户主导）
↓ 用户提出修改需求 → Agent 执行优化 → 更新 draft.md
↓ 迭代优化直到满意

阶段7: 最终输出
↓ 使用 script.md 模版 → 生成最终脚本到项目根目录
```

---

## 🔑 关键规则

### 1. 模版包含完整流程

每个模版都包含：
- ✅ 输出内容结构
- ✅ 下一步引导/确认问题
- ✅ 示例参考

所有流程信息都集成在模版文件中，Agent 读取模版即可理解完整工作流。

### 2. 两步输出模式

阶段2/4/5 使用"两步输出"：
- **第一步**：生成内容 + 请用户确认
- **第二步**：用户确认后，输出"下一步行动确认"文档

### 3. 文件管理规则

**当前版本**：
- `{PROJECT}/stages/idea.md`
- `{PROJECT}/stages/frame.md`
- `{PROJECT}/stages/research.md`（可选）
- `{PROJECT}/stages/outline.md`
- `{PROJECT}/stages/draft.md`
- `{PROJECT}/script.md`（项目根目录）

**历史版本**：
- `{PROJECT}/_archive/idea_v01.md`, `idea_v02.md`...
- `{PROJECT}/_archive/frame_v01.md`, `frame_v02.md`...
- `{PROJECT}/_archive/outline_v01.md`, `outline_v02.md`...
- `{PROJECT}/_archive/draft_v01.md`, `draft_v02.md`...

### 4. 时长和字数换算

- 1分钟 ≈ 150-180字（根据语速和内容类型）
- 教育类内容偏多（180字/分钟）
- 故事类内容偏少（150字/分钟）

**时长显示规则**：
- <60秒用秒（如"30秒"）
- ≥60秒用分钟（如"1.5分钟"）

### 5. 平台适配说明

⚠️ **多平台适配不在模版中处理**

所有 `stage/*.md` 模版只生成**单一目标平台**的内容。多平台适配（YouTube/B站/抖音/小红书）在 CLAUDE.md 的"阶段5: 脚本撰写"阶段通过"平台智能适配机制"处理。

---

## 📖 设计说明

本模版系统经过充分的实践验证和迭代优化，能够支持从选题到最终脚本输出的完整工作流。

所有模版都遵循以下设计原则：
- **自包含性**：每个模版包含完整的输出结构和引导信息
- **流程自然**：通过"下一步确认"机制自然推进工作流
- **灵活适配**：支持不同视频类型和时长的创作需求
- **版本控制**：完善的文件归档机制保证历史版本可追溯

---

## 🚀 使用方式

**Agent 使用流程**：
1. 读取 `.claude/CLAUDE.md` 了解阶段流程
2. 读取 `.claude/template/stage/{stage-name}.md` 获取输出模版
3. 按照模版结构生成输出
4. 模版自带"下一步引导"，自然推进流程
